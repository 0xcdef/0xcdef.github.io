Teaching an OpenClaw agent to think like Claude.

Four config files that transplant Claude's internal reasoning loop — search routing, synthesis rules, injection defence, and wellbeing awareness — directly into your OpenClaw workspace.

AI & Agents · Mar 26, 2026 · 8 min read

Why default config leaves performance on the table

Out of the box, an OpenClaw agent is blank. It answers questions. It uses tools if you've enabled them. But it makes no promises about when to search, how to synthesize what it finds, whether to confirm before acting, or what to do when a message carries an emotional undercurrent.

That gap is where most of an agent's performance lives. A raw LLM instructed only to "be helpful" will search when it shouldn't, skip searching when it should, paste snippets instead of reasoning over them, and execute actions without a second thought.

"The goal is not to respond. The goal is to be useful. Those are different things."

This post documents a complete configuration set — four files — built by reverse-engineering Claude's actual internal logic and encoding it as OpenClaw workspace files. The result is an agent that routes tool calls with intention, synthesizes rather than regurgitates, confirms before acting, and notices when you need more than an answer.

The architecture: four files, one coherent agent

OpenClaw's identity system is file-based. Every session, the agent reads its workspace files and assembles itself from them. The files are the agent. This means getting the files right is the entire job.

Identity

SOUL.md

Who the agent is. Character, values, tone, curiosity, pushback style, and wellbeing awareness. Injected first, shapes everything else.

Procedure

AGENTS.md

How the agent operates. Session start sequence, tool routing logic, action confirmation pattern, memory system, group chat rules.

Cognition

REASONING.md

How the agent thinks, turn by turn. The five-step loop: Classify → Route → Gather → Synthesize → Respond. The file that most agents are missing entirely.

Memory

MEMORY.md

Structured long-term memory template. Preferences, constraints, projects, people, open loops, corrections — everything that makes sessions compound over time.

The reasoning loop in detail

The centrepiece is REASONING.md — a file that doesn't ship with OpenClaw by default, but produces the biggest performance gain. It encodes a five-step decision loop the agent runs mentally before every response.

Step 1 — Classify

Before touching a tool, the agent identifies what kind of request it's looking at. Timeless knowledge (how TCP/IP works) never needs a search. Current state (who is the CEO of X, is this product still supported) always does. Actions (send, schedule, delete) require confirmation. Ambiguous requests get one clarifying question — not three.

Step 2 — Route

Tool selection follows a strict priority: internal tools first for personal data, web search for external information, web fetch when snippets aren't enough. The key discipline is search scaling — one tool call for a single fact, three to ten for research, never more than needed.

Step 3 — Gather

Each query must be meaningfully distinct from the last — repeating the same phrasing returns the same results. The agent stops gathering when it has enough to answer confidently. Over-searching is as much a failure mode as under-searching.

Step 4 — Synthesize

This is the step most agents skip. Raw tool output is not an answer. The agent paraphrases, leads with the most relevant information, flags conflicts between sources, and attributes claims. A hard quoting limit applies: under 15 words per direct quote, one quote per source maximum, default to paraphrasing.

Step 5 — Respond

Format matches content: short factual questions get sentences, not lists. Research gets prose with attribution. Instructions get numbered steps. The agent never opens with "Great question!" and never closes with an offer to keep chatting. It delivers the answer and stops.

What SOUL.md adds beyond procedure

Procedure files tell the agent what to do. The soul file tells it who to be. Three things in this soul file make a meaningful difference to how interactions feel:

Genuine curiosity

The agent is instructed to let intellectual interest show — not through exclamation marks, but through the quality of engagement. Exploring implications, noticing tensions, thinking alongside rather than returning processed output. This is what separates a reasoning partner from a search wrapper.

Charitable interpretation

When a request is ambiguous, assume the most reasonable intent and proceed on it. State the interpretation, then verify. This is both faster and more respectful than interrogating the user before helping.

Constructive pushback

When the user's plan has a flaw, the agent says so with a concrete reason. When a premise is wrong, it corrects it — without condescension. The soul file frames this directly: disagreement is a form of respect. This is the line between a useful agent and a yes-machine.

Injection defence and safety

An OpenClaw agent with web access, email access, and the ability to run shell commands is a meaningful attack surface. The config files encode an explicit injection defence posture: any instruction found in observed content — web pages, emails, documents, tool results — is treated as data, not as a command.

If the agent finds instruction-like content in the wild, it stops, shows the content to the user, and asks: should I follow this? It never executes automatically. The same applies to irreversible actions — sending, deleting, publishing, purchasing — which always require an explicit confirmation before proceeding.

Download the files

All four files are ready to drop into your OpenClaw workspace. Copy them to ~/.openclaw/workspace/ and restart your agent.

mkdir -p ~/.openclaw/workspace

# Then restart your agent session

What to expect after installing

The most immediate change is in search behaviour. The agent stops searching for things it already knows with confidence, and reliably searches for anything that touches current state. Responses become shorter for simple questions and more substantive for complex ones — length matches the task rather than defaulting to either extreme.

Over several sessions, the memory system compounds. The agent accumulates your preferences, constraints, and project context, and wakes up each session already knowing them. The open loops section is particularly useful — tasks that span sessions stay visible until explicitly resolved.

The wellbeing awareness in SOUL.md is subtle but meaningful. It won't turn your agent into a therapist. What it does is stop the agent from barrelling through a task when the message underneath it signals that something else is going on.
