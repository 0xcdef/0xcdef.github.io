Claude Setup · Prompt Engineering

Your Claude setup is working against you.

More rules don't mean better outputs. Here's how to audit your entire Claude configuration in one message — and cut it down to what actually works.

AI AGENTS · MAR 24, 2026 · 5 min read

The rule-stacking trap

It starts innocently. You get a bad output, so you add a rule: "be more concise." Next week, another bad output, another rule: "use a casual tone." A month later: "always explain technical terms."

Each addition feels productive — you're fixing real problems as they come up. But three months in, you've got 30 rules piled on top of each other, and the model is trying to follow all of them at once, which means it's doing none of them well.

"It's like handing a chef a 47-step recipe when they only need 12. The extra 35 steps slow them down, make them second-guess the parts they already know, and the dish comes out worse."

That's what over-prompting does. Anthropic's own engineering team discovered this while building Claude Code — their scaffolding was making the AI worse. If it happens at Anthropic, it's almost certainly happening in your custom setup too.

Why rules go stale

Not all bad rules are obviously bad. They fall into predictable categories that accumulate silently over time.

Silent Waste

Default behavior

Rules that tell Claude to do things it already does. "Be professional," "don't make things up," "use proper grammar." These add noise without adding signal.

⚠ Most common type — easy to miss
Active Harm

Conflicting rules

"Be concise" fighting "always explain your reasoning." "Use a casual tone" colliding with "maintain professional language." These force the model to guess which rule wins.

⚠ Degrades output unpredictably
Clutter

Redundant rules

The same instruction written three different ways across three different files. Each copy dilutes the others and makes the whole setup harder to reason about.

✓ Easy to remove once spotted
Reactive Fixes

One-off patches

Rules added to fix a single bad output that never recurred. "Don't use the word 'delve'." These address symptoms, not patterns, and rarely generalize.

✓ Usually safe to delete

The one-message audit

Instead of manually reading through your setup line by line, tell Claude to audit itself. In Claude Code or Cowork, Claude has direct access to your claude.md, skills folder, context files — everything. Use this prompt:

# Paste this into Claude Code or Cowork Read my entire setup before responding. Check my claude.md, every skill in my skills folder, every file in my context folder, and any other instruction files you can find. Then go through every rule, instruction, and preference you found. For each one, tell me: 1. Is this something you already do by default? 2. Does this contradict another rule elsewhere in my setup? 3. Does this repeat something already covered elsewhere? 4. Does this look like it was added to fix one specific bad output? 5. Is this so vague you'd interpret it differently every time? Then give me: - A list of everything you'd cut, with a one-line reason each - A list of any conflicts found between files - A cleaned-up version of my claude.md with dead weight removed

One message. Claude reads your entire setup, audits it, and returns exactly what to cut and why — no manual file digging required.

Five questions for every rule

Whether you run the audit prompt or review manually, apply the same five questions to every instruction you find.

QuestionFlag if...Action
Does Claude already do this by default? Yes Cut it
Does this conflict with another rule? Yes Resolve or cut one
Is this covered somewhere else? Yes Keep the clearest copy
Was this added to fix a single bad output? Likely Validate then cut
Is this too vague to be consistent? Yes Sharpen or cut

Validate before you delete

Don't blindly delete everything the audit flags. Run a quick validation loop first — it takes five minutes and prevents regressions.

The validation process

After getting the audit results, delete the flagged rules. Then run your three most common tasks with the trimmed setup. If the output is the same or better, the deleted rules were dead weight. If something specific breaks, add back just that one rule — not the whole batch.

What good looks like

A healthy setup has rules that are global (not reactive), specific (not vague), and non-contradictory. Each rule should improve every output, not just defend against one past failure.

Addition by subtraction

The goal is the minimum viable setup — the simplest configuration that produces the outputs you want. Not the most comprehensive one. Not the most defensively complete one.

A good Claude setup gets simpler over time, not longer. Every time you add a rule, ask whether you're improving the system or just patching a symptom. The ones that survive that question are the ones worth keeping.

Get the skill

The Prompt Auditor is a plain-text Markdown skill file. Save it as SKILL.md inside a prompt-auditor/ folder in your skills directory and Claude will automatically use it.
